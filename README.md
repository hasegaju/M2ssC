# SCAN LOG — バーコード読取・蓄積デモ

iPhone + Bluetoothリーダー(HIDモード)で動く、完全オフラインのデモページ。
サーバー処理なし。HTML 1枚 + Service Worker のみ。外部CDN参照ゼロ。

## ファイル構成

| ファイル | 役割 |
|---|---|
| index.html | 本体(CSS/JS すべて内包) |
| zxing.min.js | バーコード解析ライブラリ(ローカル同梱・CDN不使用) |
| sw.js | Service Worker(オフラインキャッシュ) |
| manifest.json | PWA定義(ホーム画面追加用) |
| icon-180.png / icon-512.png | アイコン |

## 設置手順(一度だけ)

1. 4ファイルを社内Webサーバーの適当なディレクトリに置く
   例: `/var/www/html/scandemo/` → `https://サーバー/scandemo/`
   ※ Service Worker の制約上 **HTTPS必須**(または http://localhost)
2. 社内Wi-Fiに繋いだ iPhone の Safari でそのURLを開く
3. 一度全部読み込まれれば Service Worker がキャッシュ完了
4. 共有ボタン →「ホーム画面に追加」
5. **機内モードにして**ホーム画面のアイコンから起動 → 動けば準備完了

以降、客先では通信不要。機内モード + Bluetooth ON だけで動きます。

## リーダー側の設定

- **HIDモード(キーボードエミュレーション)** でペアリング
- サフィックスを **Enter(CR)** に設定(大半のリーダーは初期値でEnter付き)
- ペアリング中はソフトウェアキーボードが出ません。手入力が必要なら
  リーダーのキーボード呼び出しボタン(機種による)を使うか、一時的に電源OFF

## カメラ読取

- スキャン入力欄の 📷 ボタンでカメラ起動(背面カメラ優先)
- 対応フォーマット: JAN/EAN-13, EAN-8, CODE-128, CODE-39, ITF, QR
- 初回はカメラ許可のダイアログが出る → **設置後の動作確認時に一度許可しておくこと**
  (ホーム画面追加後のPWAは許可状態が別扱いになる場合があるため、
   必ず「ホーム画面から起動した状態」で一度カメラを試しておく)
- 同一コードは2.5秒間再読取しない(連射防止)。別コードは連続スキャン可
- カメラも getUserMedia の制約で **HTTPS必須**(Service Workerと同条件)

## カスタマイズ

- **初期サンプルデータ**: index.html 内の `SEED` 配列を編集(空配列 `[]` で無しに)
- **タイトル/名称**: `<title>` とヘッダーの `SCAN LOG` を書き換え
- **更新の反映**: index.html を修正したら sw.js の `VERSION` を `scanlog-v2` 等に
  上げる → オンライン状態で一度開き直すとキャッシュが更新される

## 機能メモ

- JAN-13 / JAN-8 はチェックディジットを検証して緑バッジ表示。それ以外は CODE 扱い
- 同一コードを再スキャンすると数量が +1(重複バッジ表示)
- CSV書き出しは UTF-8 BOM付き(Excelで文字化けしない)。iPhoneでは共有シートが開く
- データはメモリ上のみ。ページを閉じるとリセット(次のデモは毎回まっさら)
